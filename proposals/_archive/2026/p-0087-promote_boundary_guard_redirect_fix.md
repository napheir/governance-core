# Proposal P-0087: Promote issue #20: boundary-guard read-only fast-exit must not skip write-redirect targets

## Trigger

User asked to run the full curation flow on candidate issue #20 (mechanism,
from consumer `trade-agent`). The payload patches `session-boundary-guard.py`,
a security-sensitive guard source. Per `/proposal classify` this is
PROPOSAL_REQUIRED: it touches a hook/guard source, is security-sensitive, and
needs a rollback path. Curating a capability change is governed by
`/curate-candidate` step 8 (formalize via `/proposal`).

## Scope

Promote the issue #20 fix into the **package source** (`governance_core/`),
single authoritative source per Art.11.1:

- `governance_core/tools/session-boundary-guard.py` — `is_read_only_bash()`
  gains a redirect short-circuit: when a file-write redirect (`>` / `>>`,
  excluding fd-dup `2>&1` / `>&2` via negated char class `[^\s&|;<>]`) is
  present, return `False` so the command is NOT fast-exited as read-only.
  This closes a real, currently-reproducible escape where
  `cat <in> > <out-of-boundary>` — including `> ~/.ssh/...` (a critical
  path) — bypassed both path extraction and the critical-path check.
- `governance_core/tools/test_session_boundary_guard.py` — +4 regression
  cases (22 cat>outside block, 23 cat>~/.ssh critical block, 24 cat>inside
  allow, 25 `2>&1` fd-dup allow); total 21 → 25.
- `pyproject.toml` — version bump (ships to consumers via `upgrade`).
- `maintainer/consumer_registry.json` — record curation decision `promoted`.

The full-file payload was diffed against current source
(`git diff --no-index --ignore-cr-at-eol`): the ONLY material delta is the
10-line guard addition + the 4 test cases. No other drift.

## Non-Goals

- No change to the redirect detection used by `extract_bash_paths` /
  `BASH_PATH_PATTERNS` (the existing redirect extractor stays as-is; the fix
  only stops the read-only fast-exit from pre-empting it).
- No broadening of `CRITICAL_PATH_PATTERNS` or boundary semantics.
- No autonomy-layer edits (Art.11.2): root `tools/` copy is regenerated by
  `upgrade`, never hand-edited.
- Not de-trade-ified — the mechanism and its examples are already generic
  (`/outside`, `~/.ssh`); no `trade-agent` domain leakage to genericize.

## Rollback / Recovery

- Pre-archive: `git checkout -- governance_core/tools/session-boundary-guard.py
  governance_core/tools/test_session_boundary_guard.py pyproject.toml` reverts
  the working tree; the change is a single self-contained commit.
- Post-commit: `git revert <hash>` restores the prior guard; `upgrade`
  re-derives the autonomy-layer copy. The fix is purely additive (one early
  `return False` branch) — reverting cannot corrupt unrelated guard logic.
- The consumer (`trade-agent`) keeps its local patch regardless; revert only
  affects the upstream source.

## Risks

- **Over-block legitimate in-boundary redirects** (low prob / low impact):
  `cat a > b` inside the boundary now goes through path extraction instead of
  fast-exit. Mitigation: case 24 asserts in-boundary redirect still passes;
  the redirect target is checked and allowed when in-boundary.
- **fd-dup false positive** (low / low): `2>&1` could be misread as a file
  write. Mitigation: negated char class `[^\s&|;<>]` requires a real path char
  after `>`; case 25 asserts `2>&1` stays read-only.
- **Performance** (negligible): one extra `re.search` per Bash command before
  the read-only loop. Single regex, bounded command length.
